# ROLES — распределение ответственности на запуске

> Кто за что отвечает в боевом проекте. Под каждой ролью — проверяемые артефакты.

## 1. Product Owner (вы / руководитель)

**Цель:** продукт решает задачу клиента, метрики в плане.

- [ ] Утвердить оффер Hero + VARIANT.heroSubtitle для каждого из 6 сайтов (файл `lib/variants.ts`).
- [ ] Утвердить тарифы в `components/Pricing.tsx` + JSON-LD Product.price.
- [ ] Назначить менеджера по продажам, которому идут лиды (Telegram + email).
- [ ] Еженедельный ретро по воронке: лиды → квалификация → договор → выручка.

**Метрики:** лидов/неделя, CPL, CR лид→договор, средний чек.

## 2. Юрист

**Цель:** сайт не собирает ни одной жалобы в РКН/прокуратуру.

- [ ] Проверить `app/privacy/page.tsx` — редакция актуальна, оператор, ИНН, ОГРН, адрес совпадают с уставными.
- [ ] Подать уведомление в Роскомнадзор (до первого лида!) на `https://pd.rkn.gov.ru/`.
- [ ] Зафиксировать номер в реестре РКН → записать в `components/constants.ts::BRAND.rknRegistryNumber`.
- [ ] Проверить текст согласия в форме (`components/ContactForm.tsx`): цели, состав, срок, право отзыва.
- [ ] Если появится договор-оферта — добавить страницу `/offer` по той же схеме что `/privacy`.
- [ ] Сохранить чек-лист: «сайт принимает ПДн → есть согласие → есть Политика → есть уведомление в РКН → есть Модель угроз».

**Артефакты:** скан уведомления РКН, Политика ПДн, внутренний приказ об Операторе.

## 3. ИБ-инженер (НПК «Оборон-Экран»)

**Цель:** инфраструктура сайта не станет точкой компрометации компании.

- [ ] Проверить nginx CSP — в DevTools Console нет блокировок.
- [ ] Проверить `curl -I https://gisprof.ru/` — все 6 security headers (HSTS, CSP, X-Frame-Options, X-Content-Type-Options, Referrer-Policy, Permissions-Policy).
- [ ] Проверить TLS: `https://www.ssllabs.com/ssltest/analyze.html?d=gisprof.ru` — оценка A или A+.
- [ ] Проверить, что `.env`/`.git`/`composer.json` недоступны снаружи (nginx `location ~ /(...)` блокирует).
- [ ] Настроить `/etc/fail2ban/jail.local` на nginx-log (брут-форс /api/lead).
- [ ] Настроить бэкапы: `/home/deploy/sites`, `/var/log/gisprof`, `/etc/letsencrypt` → S3 / Я.Диск раз в сутки.
- [ ] Раз в 30 дней: `openssl s_client -connect gisprof.ru:443 -servername gisprof.ru < /dev/null` (срок сертификата).
- [ ] Раз в 90 дней: обновление OS (`apt update && apt upgrade`), ротация SSH-ключей.

**Артефакты:** отчёт SSL Labs, конфиг fail2ban, расписание бэкапов.

## 4. DevOps

**Цель:** деплой занимает < 5 минут, rollback < 2 минут.

- [ ] Документировать пайплайн деплоя в `docs/DEPLOY.md` (как запустить `bash scripts/server/remote-bootstrap.sh`, как обновить один сайт).
- [ ] Настроить cron на `certbot renew --quiet` (ставит `remote-bootstrap.sh`).
- [ ] Настроить cron на `logrotate` для `/var/log/gisprof/leads.jsonl` (размер 10 MB / 30 дней).
- [ ] Подключить Uptime-мониторинг (Uptimerobot, Betterstack): 6 хостов + `/api/lead` (POST /ping — просто проверка 400 response).
- [ ] Алерт в Telegram при падении любого домена.
- [ ] Смотреть дисковое пространство: `df -h` (2 GB VPS заполняется логами).

**Артефакты:** `docs/DEPLOY.md`, скриншот Uptimerobot, cron listing.

## 5. Маркетолог

**Цель:** каждый рубль в рекламе даёт отслеживаемого лида.

- [ ] Настроить цели в Яндекс.Метрике (ID из `VARIANT.metrikaId` для каждого кластера):
  - `form-submit-click` — клик по «Отправить заявку»
  - `form-submit-success` — успешная отправка (через `data-goal` + reachGoal)
  - `phone-tap-hero`, `phone-tap-hero-side`, `phone-tap-contact`, `phone-tap-footer`, `phone-tap-form` — клики по телефону
  - `email-tap`, `email-tap-hero-side`, `email-tap-footer`, `email-tap-form` — клики по e-mail
  - `hero-check`, `hero-contact`, `header-contact` — CTA в Hero и Header
- [ ] Настроить UTM-метки в Яндекс.Директ: `?utm_source=yandex&utm_medium=cpc&utm_campaign=gis-prof-spb-hot&utm_term={keyword}`.
- [ ] Проверить, что UTM доходят до лида: в `/var/log/gisprof/leads.jsonl` поле `utm` заполнено.
- [ ] Раз в неделю — отчёт по воронке: показы → клики → CTR → лиды → CPL.
- [ ] A/B оффера Hero: тестировать `VARIANT.h1Accent` через A/B в Директе (2 разные кампании на 2 разных домена).

**Артефакты:** скриншот целей в Метрике, отчёт CPL/неделя.

## 6. Контент-менеджер

**Цель:** свежий трафик из Яндекса каждый месяц.

- [ ] Раз в неделю — обновление FAQ (`components/Faq.tsx`), 1 новый вопрос.
- [ ] Раз в месяц — отзыв (`components/Reviews.tsx`), реальный от клиента.
- [ ] Раз в 2 месяца — кейс (`components/Cases.tsx`) с метриками до/после.
- [ ] Подавать IndexNow ping после обновлений: `curl -X POST 'https://yandex.com/indexnow?url=https://gisprof.ru/&key=<INDEXNOW_KEY>'`.
- [ ] Следить за упоминаниями «ГИС Профилактика» в Яндекс.Новостях, быстро реагировать (новая редакция 411-ПП, поправки в КоАП и т.д.).

**Артефакты:** редплан, IndexNow лог пингов.

## 7. Продажник

**Цель:** перезвонить лиду за 15 минут, квалифицировать, выставить КП.

- [ ] Telegram-бот `@<gisprof_leads_bot>` подключён, все лиды падают в чат.
- [ ] Скрипт первого звонка (15 мин): выяснить число АРМ, класс ИСПДн, дедлайн, бюджет.
- [ ] Ответ по Whatsapp/Telegram в рабочее время ≤ 15 минут.
- [ ] CRM (Битрикс24 / Мегаплан / AmoCRM): все лиды заведены; источник = вариант домена.
- [ ] Еженедельно — отчёт воронки: лид → квалифицирован → КП → договор → оплата.

**Артефакты:** Telegram-чат с лидами, воронка CRM.

## 8. Тех.писатель / поддержка

**Цель:** клиент знает «что делать сразу после договора», сотрудники знают «как обновлять сайт».

- [ ] Актуализировать `README.md` раз в квартал (версии зависимостей, node version).
- [ ] Поддерживать `docs/RELEASE-CHECKLIST.md` и `docs/ROLES.md` (этот файл).
- [ ] При первом внедрении у клиента — снимать видео, текстовая инструкция → кейс (обезличенный).

## Ролевая матрица RACI (summary)

| Задача | PO | Юрист | ИБ | DevOps | Маркет | Контент | Sales |
|---|---|---|---|---|---|---|---|
| Сайт запущен | A | C | R | R | I | I | I |
| Лид пришёл и обработан | A | I | I | I | R | I | R |
| РКН уведомление подано | C | **R/A** | I | I | I | I | I |
| SSL обновлён | I | I | R | **A** | I | I | I |
| Яндекс.Метрика считает | I | I | I | C | **R/A** | I | I |
| Цена в договоре = цена на сайте | **R/A** | C | I | I | C | I | C |
| Новая заявка → звонок ≤ 15 мин | A | I | I | I | I | I | **R** |

Обозначения: **R** — исполнитель, **A** — ответственный (принимает решение), C — консультирует, I — информируется.
